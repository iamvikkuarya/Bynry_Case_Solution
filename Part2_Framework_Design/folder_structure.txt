KEY PRINCIPLES:
==============

1. Separation of Concerns
   - Tests only contain test logic
   - Pages contain element locators and interactions
   - Config contains all configuration
   - Utils contain reusable helpers

2. DRY (Don't Repeat Yourself)
   - Common code in base classes
   - Reusable utilities
   - Shared fixtures

3. Easy to Navigate
   - Clear folder names
   - Organized by test type (ui, api, mobile)
   - Everything has its place

4. Scalable
   - Easy to add new pages
   - Easy to add new tests
   - Easy to add new environments
